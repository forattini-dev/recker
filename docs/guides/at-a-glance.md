# Recker at a Glance

> A quick overview of Recker's features and capabilities.

## Why Recker?

**The HTTP Client Built for the AI Era** - Recker provides developer-friendly APIs with first-class AI integration support, combining performance with excellent developer experience.

## Core Features

| Feature | Description |
|---------|-------------|
| **19 HTTP Methods** | Full HTTP spec support including WebDAV and CDN methods |
| **HTTP/2 Support** | Native HTTP/2 with automatic protocol negotiation |
| **TypeScript First** | Written in TypeScript with full type safety |
| **Smart Retry** | Exponential, linear, and decorrelated backoff with jitter |
| **Circuit Breaker** | Protect your services from cascading failures |
| **SSE Streaming** | Built-in Server-Sent Events parsing for AI/LLM apps |
| **WebSocket** | Full-featured WebSocket client with reconnection |
| **Hooks System** | beforeRequest, afterResponse, onError hooks |
| **Auto Compression** | Automatic gzip/brotli handling |
| **Built-in Caching** | Memory, File, and Redis storage with SWR support |
| **Rate Limiting** | Smart rate limit detection and handling |
| **DNS/WHOIS** | Built-in network utilities |
| **Concurrency Control** | Global limits, per-batch control, connection pooling |
| **HTML Scraping** | jQuery-like API with Cheerio integration |
| **GraphQL** | First-class GraphQL support with error handling |

## When to Choose Recker

Choose Recker if you need:

- **AI Integration** - SSE streaming, structured error handling for LLMs
- **Built-in Resilience** - Circuit breakers, smart retries, concurrency control
- **Performance** - HTTP/2, connection pooling, automatic compression
- **Developer Experience** - Type safety, rich observability, detailed timings
- **Network Utilities** - DNS, WHOIS, header parsing built-in
- **WebSocket Support** - Without adding separate libraries
- **HTML Scraping** - Extract data from websites with ease
